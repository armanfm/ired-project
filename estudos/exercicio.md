# Plano de 30 Dias – Treino de Solidity Sem Dependência de IA

## Semana 1 – Fundamentos Sólidos
**Objetivo:** Conseguir escrever funções básicas sozinho.

| Dia | Foco | Exercício |
|-----|------|-----------|
| 1–2 | Variáveis, tipos, operadores | - Contrato que armazena e retorna um número<br>- Contrato que armazena e retorna um texto<br>- Contrato que converte uint para string |
| 3–4 | Estruturas de controle (if, else, loops) | - Contrato que calcula bônus de saldo com if/else |
| 5–6 | Funções e visibilidade (public, private) | - Funções que somam, subtraem ou alteram variáveis |
| 7 | Revisão prática | - Combine variáveis, controle e funções em um contrato simples |

---

## Semana 2 – Contratos Básicos
**Objetivo:** Escrever contratos simples sem olhar vídeos.

| Dia | Foco | Exercício |
|-----|------|-----------|
| 8–9 | Contratos e deploy local (Remix/Foundry) | - Criar contrato `Wallet` que armazena saldo de usuário |
| 10–11 | Arrays e Mapeamentos | - Criar lista de usuários e mapping de saldo |
| 12–13 | Eventos e logs | - Adicionar eventos, ex.: `TransferenciaRealizada` |
| 14 | Mini projeto | - Wallet simples com array, mapping e eventos; funções de depósito e saque |

---

## Semana 3 – Tokens e Padrões
**Objetivo:** Aplicar padrões reais do mercado.

| Dia | Foco | Exercício |
|-----|------|-----------|
| 15–16 | ERC20 básico | - Criar token simples: nome, símbolo, supply inicial |
| 17–18 | ERC721 básico (NFT) | - Criar NFT básico sem metadata complexa |
| 19–20 | Funções de transferência e segurança | - Adicionar `require` para controle de transferência |
| 21 | Revisão prática | - Combinar ERC20 ou ERC721 com Wallet simples |

---

## Semana 4 – Desafios e Projetos
**Objetivo:** Consolidar lógica e independência.

| Dia | Foco | Exercício |
|-----|------|-----------|
| 22–23 | Ethernaut / Foundry Challenges | - Resolver 1–2 níveis focados em lógica ou segurança |
| 24–25 | Mini projeto solo | - Criar contrato próprio com depósito, saque, transferência, eventos |
| 26–27 | Testes unitários (Foundry ou Hardhat) | - Testar funções do seu contrato sem tutorial |
| 28–29 | Revisão geral | - Escrever contrato completo sozinho (Wallet + Token + eventos) |
| 30 | Deploy final e documentação | - Deploy local/testnet e escrever README explicando o contrato |

---

## Dicas Extras
1. Nunca copie código pronto – tente escrever sozinho e depois compare.  
2. Documente todos os erros – cada bug é aprendizado.  
3. Foco na lógica, não na sintaxe perfeita.  
4. Use IA apenas como coach, nunca para escrever código pronto.  
5. Sessões curtas de 30–60 min diárias funcionam melhor que maratonas.
